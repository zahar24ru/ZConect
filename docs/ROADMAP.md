# Дорожная карта реализации

## Итерация 1 — Базовая сессия и signaling

- [ ] Go signaling сервис (`/session/create`, `/session/join`, `/ws`)
- [ ] Redis интеграция (TTL сессии, attempts, lock)
- [ ] Генерация кодов 8+8
- [ ] Базовые server logs в `logs.log`
- [ ] Документация API и env

Критерий готовности:

- Можно получить коды и выполнить pairing двух клиентов через WS.

## Итерация 2 — WebRTC подключение + видео

- [ ] Установка peer connection
- [ ] ICE/STUN/TURN
- [ ] Захват экрана (single display)
- [ ] Передача видео VP8
- [ ] Профили качества LOW/MEDIUM/HIGH

Критерий готовности:

- Стабильный видео-поток в течение 10+ минут.

## Итерация 3 — Управление и UX

- [ ] Передача мыши/клавиатуры
- [ ] Опция подтверждения доступа (on/off)
- [ ] Unattended режим
- [ ] Выбор экрана/всех экранов
- [ ] UI в стиле "простая панель"

Критерий готовности:

- Полный удаленный контроль с управляемым UX.

## Итерация 4 — Clipboard + файлы

- [ ] Синхронизация буфера обмена (текст)

Критерий готовности:

- Обмен текстом и одиночным файлом между клиентами.
